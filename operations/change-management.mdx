# Change Management

This document describes change management procedures for infrastructure changes.

## Change Process

1. **Plan**: Plan the change
2. **Review**: Review with team
3. **Test**: Test in staging
4. **Deploy**: Deploy to production
5. **Verify**: Verify the change

## Best Practices

- Document all changes
- Review before deploy
- Test in staging first
- Have rollback plan
