# Glossary

This glossary defines terms and acronyms used throughout the documentation.

## A

**Agentic AI**: AI systems that can act autonomously to achieve goals, making decisions and taking actions without constant human intervention.

**API Gateway**: A service that acts as an entry point for API requests, handling routing, authentication, rate limiting, and other cross-cutting concerns.

**Auto-scaling**: The automatic adjustment of compute resources based on demand, scaling up or down as needed.

## B

**Backend (Terraform)**: The remote storage location for Terraform state files, such as Google Cloud Storage.

**Blueprint**: A reference architecture pattern that can be used as a template for deployments.

## C

**Cloud Armor**: Google Cloud's DDoS protection and web application firewall (WAF) service.

**Cloud KMS**: Google Cloud Key Management Service for managing encryption keys.

**Cloud Run**: Google Cloud's serverless container platform that automatically scales containers.

**CrewAI**: A framework for orchestrating multiple AI agents to work together on complex tasks.

**CI/CD**: Continuous Integration and Continuous Deployment - automated processes for building, testing, and deploying code.

## D

**Defense in Depth**: A security strategy that uses multiple layers of security controls.

## E

**Encryption at Rest**: Encryption of data stored on disk or in databases.

**Encryption in Transit**: Encryption of data while it's being transmitted over networks.

## F

**Firestore**: Google Cloud's NoSQL document database.

**Firewall Rule**: A network security rule that controls traffic to and from resources.

## G

**GCP**: Google Cloud Platform.

**GKE**: Google Kubernetes Engine - Google's managed Kubernetes service.

**GPU**: Graphics Processing Unit - specialized hardware for parallel processing, often used for AI/ML workloads.

## H

**High Availability (HA)**: A system design approach that ensures maximum uptime and minimal downtime.

**HTTPS**: Hypertext Transfer Protocol Secure - encrypted HTTP protocol.

## I

**IAM**: Identity and Access Management - system for managing user and service account permissions.

**IaC**: Infrastructure as Code - managing infrastructure through code rather than manual processes.

**Instance Group**: A collection of VM instances managed as a single entity for load balancing and auto-scaling.

## K

**KMS**: Key Management Service - see Cloud KMS.

## L

**Load Balancer**: A service that distributes incoming traffic across multiple backend instances.

**Least Privilege**: Security principle of granting only the minimum permissions necessary.

## M

**Managed Instance Group (MIG)**: A group of identical VM instances managed by Google Cloud for auto-scaling and load balancing.

**Module (Terraform)**: A reusable collection of Terraform resources.

**Multi-Region**: Deployment across multiple geographic regions for high availability.

## N

**Network Tag**: A label applied to VM instances for firewall rule targeting.

## O

**Organization Policy**: A policy that enforces constraints on Google Cloud resources at the organization level.

## P

**Preemptible Instance**: A VM instance that can be terminated by Google Cloud with short notice, offered at a lower price.

**Private Google Access**: Allows VM instances without external IP addresses to access Google APIs and services.

**Pub/Sub**: Google Cloud Pub/Sub - a messaging service for event-driven architectures.

## R

**Region**: A geographic location where Google Cloud resources are deployed.

**Replica**: A copy of a database or storage resource, often used for read scaling or disaster recovery.

## S

**Secret Manager**: Google Cloud service for storing and managing secrets like API keys and passwords.

**Service Account**: A special type of Google account used by applications and services to authenticate to Google Cloud APIs.

**Shielded VM**: A VM with enhanced security features including secure boot, vTPM, and integrity monitoring.

**SLO**: Service Level Objective - a target level of service reliability.

**State (Terraform)**: Terraform's record of the infrastructure it manages, mapping configuration to real resources.

**Subnet**: A subdivision of a VPC network.

## T

**Terraform**: An Infrastructure as Code tool by HashiCorp.

**TLS**: Transport Layer Security - cryptographic protocol for secure communication.

## V

**VPC**: Virtual Private Cloud - an isolated network environment in Google Cloud.

**VPC Service Controls**: Security feature that creates a perimeter around Google Cloud APIs to prevent data exfiltration.

**Vertex AI**: Google Cloud's unified AI/ML platform.

## Z

**Zero Trust**: Security model that requires verification for every access request, regardless of location.

**Zone**: A deployment area within a region. Resources in a zone are isolated from failures in other zones.
