# ü§ñ CrewAI Project Creation Blueprint for GCP

## üåü Overview

This **Terraform blueprint** provides a fully automated solution for creating a complete **GCP project** with the necessary infrastructure for a **CrewAI Enterprise Platform** deployment. It manages everything from the organizational folder structure to a production-ready Kubernetes cluster and all supporting services.

## ‚ú® What Gets Created

This blueprint deploys over **70 resources** across two main categories:

### üì¶ Project Foundation

* **GCP Project** with metadata and labels.

* **50+ Enabled APIs** for core GCP services.
* **Service Accounts** and Least-Privilege **IAM Roles**.
* **Budget Alerts** with Pub/Sub notifications and Cloud Functions for monitoring.

## üöÄ Quick Start (For Developers)

Follow these steps to deploy the entire CrewAI platform infrastructure.

<Steps>
<Step title="Navigate and Authenticate">
Navigate to the project directory and authenticate with both GCP and AWS (for the S3 backend).

```bash
# 1. Navigate to project
cd D:\Konecta\CrewAI\Project_Creation_Blueprint

# 2. Authenticate with GCP and AWS
gcloud auth application-default login
aws sso login --profile default_core_irlanda
````

\</Step\>
\<Step title="Validate Code Quality"\>
Run the `make validate` command, which automatically executes `terraform fmt`, `terraform validate`, and `tflint`.

```bash
# 3. Validate code quality (auto-format, lint, validate)
make validate
```

> **Tools Used**: This single command ensures **consistent style**, **valid syntax**, and adherence to **DevOps best practices**.

\</Step\>
\<Step title="Initialize and Plan Deployment"\>
Initialize Terraform and generate an execution plan to preview all infrastructure changes.

```bash
# 4. Initialize Terraform and preview changes
make init
make plan
```

\</Step\>
\<Step title="Deploy Infrastructure"\>
Apply the plan to provision all 70+ resources in your GCP project. This process takes approximately 20 minutes.

```bash
# 5. Deploy infrastructure (70 resources, ~20 minutes)
make apply
```

\</Step\>
\</Steps\>

**That's it\!** üéâ The `Makefile` automates all DevOps best practices, ensuring a robust deployment.

## üîí Security Highlights

The infrastructure is **Secure by Default** with a focus on least privilege and private networking.

  * üîí **Private GKE Nodes** - No public IPs on worker nodes
  * üîí **Private Cloud SQL** - Access restricted via **VPC peering** only
  * üîí **Authorized Networks** - Terraform runner IP is **auto-detected** for kubectl access
  * üîí **Encryption at Rest** - Applied to GCS, Cloud SQL, and GKE disks
  * üîí **Secret Manager** - Database credentials stored securely
  * üîí **Least-Privilege IAM** - Minimal permissions granted to all Service Accounts

## ‚öôÔ∏è Prerequisites and Configuration

### Required Tools

Ensure you have the following installed on your local machine:

  * **Terraform** (\>= 1.0.0)
  * **TFLint** (\>= 0.60.0)
  * **Make**
  * **gcloud CLI**
  * **AWS CLI**

### Required Permissions

  * GCP Organization/Folder Admin role
  * Billing Account Admin role
  * AWS credentials for the S3 backend (`default_core_irlanda` profile)

### Configuration

Edit the **`terraform.auto.tfvars`** file to set required organization and project details.

```hcl

# Budget
```
